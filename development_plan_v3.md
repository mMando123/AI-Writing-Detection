## 🧠 كيف تكشف أدوات مثل ZeroGPT النص؟

### المقاييس الثلاثة الأساسية

```
┌──────────────────────────────────────────────────┐
│          خوارزمية الكشف (GPTZero/ZeroGPT)        │
├──────────────────────────────────────────────────┤
│ 1. Perplexity (الحيرة)                           │
│    → هل كل كلمة "متوقعة" بعد سابقتها؟           │
│    → AI = حيرة منخفضة (متوقع جداً)              │
│    → Human = حيرة مرتفعة (مفاجآت لغوية)         │
│                                                  │
│ 2. Burstiness (الانفجارية)                       │
│    → هل التباين في الجمل ثابت أم متذبذب؟         │
│    → AI = تباين منخفض (جمل متشابهة الطول)       │
│    → Human = تباين عالٍ (قصيرة ثم طويلة)        │
│                                                  │
│ 3. Stylometric Patterns (البصمة الأسلوبية)       │
│    → عبارات انتقالية نمطية, رسمية ثابتة          │
│    → غياب الصوت الشخصي والأخطاء الطبيعية        │
└──────────────────────────────────────────────────┘
```

### لماذا الاستبدال الحالي لا يكفي؟

| الأسلوب | ما يغيّره | ما لا يغيّره |
|---------|----------|-------------|
| استبدال مرادفات | كلمات فردية | **بنية الجملة**, التدفق, الإيقاع |
| إزالة بصمات | عبارات نمطية | **التوزيع الاحتمالي**, تجانس الأطوال |
| إضافة hedges | بداية بعض الجمل | **البنية العميقة**, نمط التفكير |

> **الخلاصة:** أدوات الكشف الحديثة تحلل **التوزيع الاحتمالي على مستوى الجملة الكاملة** — تغيير كلمة هنا وهناك لا يغيّر هذا التوزيع بشكل كافٍ.

### المرحلة 2: إعادة الصياغة الهيكلية (أولوية عالية — 3-5 أيام)

> **الفكرة الأساسية:** بدلاً من تغيير كلمات، نغيّر **بنية الجملة بالكامل**.

#### 2.1 تحويلات بنيوية (Syntactic Transformations)

```python
# أمثلة على التحويلات المطلوبة:

# 1. Active → Passive (والعكس)
"The university promotes cultural diversity"
→ "Cultural diversity is promoted by the university"

# 2. Clause Fronting (تقديم الجملة الفرعية)
"Students face challenges because they are in a new environment"
→ "Being in a new environment, students face challenges"

# 3. Nominalization (تحويل الفعل لاسم)
"We analyzed the data carefully"
→ "A careful analysis of the data was conducted"

# 4. De-nominalization (تحويل الاسم لفعل)
"The implementation of the policy was successful"
→ "The policy was implemented successfully"

# 5. Sentence Combining (دمج ذكي)
"Islam values diversity. It encourages learning about others."
→ "Islam values diversity and encourages learning about others."

# 6. Sentence Splitting (تقسيم ذكي)
"The student, who was struggling with bias, found support in the community."
→ "The student was struggling with bias. They found support in the community."
```

#### 2.2 التنفيذ التقني
- [ ] **استخدام spaCy** لتحليل بنية الجملة (Dependency Parsing)
  ```
  pip install spacy
  python -m spacy download en_core_web_sm
  ```
- [ ] **بناء `SyntacticTransformer` class** يحتوي على:
  - `active_to_passive(sent)` — تحويل المعلوم للمجهول
  - `passive_to_active(sent)` — العكس
  - `front_clause(sent)` — تقديم الجملة الفرعية
  - `nominalize(sent)` — تحويل فعل لاسم
  - `combine_sentences(sent1, sent2)` — دمج جملتين
  - `split_complex(sent)` — تقسيم جملة معقدة

#### 2.3 قاموس التحويلات الذكية
- [ ] بناء **قاموس أنماط مع قواعد regex** لكل تحويل
- [ ] **احتمالية التطبيق عشوائية** — لا نطبق نفس التحويل دائماً
- [ ] **تجنب التحويل إذا كسر المعنى** — فحص طول الناتج

### المرحلة 3: أنسنة عميقة (أولوية عالية — 3-5 أيام)

#### 3.1 حقن "الصوت البشري" (Human Voice Injection)

**المشكلة الجوهرية:** نصوص AI لها صوت "محايد" بدون شخصية. أدوات الكشف تبحث عن هذا النمط تحديداً.

**التقنيات المطلوبة:**

| التقنية | المثال | التأثير |
|---------|--------|---------|
| **تعليقات شخصية** | "I've seen this firsthand during my studies" | يكسر الحيادية |
| **تردد وتراجع** | "Well, this might sound odd, but..." | يحاكي التفكير البشري |
| **تقييد الثقة** | "as far as I can tell" / "I think" | يقلل اليقين الآلي |
| **ربط بتجارب** | "Something similar happened at my campus" | يضيف سياق فريد |
| **أسئلة حقيقية** | "But does this really work in practice?" | يكسر التدفق الخطي |
| **تصحيح ذاتي** | "or rather, I should say..." | يحاكي المسودة البشرية |

#### 3.2 تنويع الإيقاع (Rhythm Variation)
- [ ] **خوارزمية "الموجة":** التأكد أن أطوال الجمل تتذبذب بنمط عشوائي
  ```
  مثال — نمط AI:    15, 14, 16, 15, 14 (متجانس)
  مثال — نمط بشري:  5, 22, 8, 30, 3, 18 (متذبذب)
  ```
- [ ] **حد أقصى لتتابع الجمل المتشابهة:** إذا كانت 3 جمل متتالية بنفس الطول ± 3 كلمات → نقسم واحدة أو ندمج اثنتين

#### 3.3 إزالة "التماسك المفرط" (Over-Coherence)
**المشكلة:** AI يربط كل جملة بسابقتها بروابط منطقية (Furthermore, Additionally, Moreover). البشر أحياناً يقفزون بين أفكار.

- [ ] **حذف 30-40% من الروابط الانتقالية** واستبدالها بـ:
  - بداية مباشرة (بدون رابط)
  - سؤال
  - عبارة شخصية
- [ ] **إضافة "استطرادات" قصيرة** بين بعض الفقرات (tangential remarks)

### المرحلة 4: تكامل خارجي + تحسين مستمر (اختياري — أسبوع+)

#### 4.1 تكامل مع LLM API (الأقوى والأكثر فعالية)

> **هذه هي الطريقة الأكثر فعالية** لتجاوز الكشف — لكنها تتطلب مفتاح API.

**الفكرة:** بدلاً من تعديل النص بقواعد ثابتة، نرسل كل فقرة إلى LLM (مثل GPT-3.5 أو Gemini) مع prompt ذكي يطلب إعادة صياغة بأسلوب بشري محدد.

```python
# مثال على الـ Prompt المقترح:
REWRITE_PROMPT = """
Rewrite the following paragraph as if YOU are a real student 
writing a case study assignment. Requirements:
- Use first person ("I", "my") naturally
- Include 1-2 personal observations or opinions
- Vary sentence lengths dramatically (mix 5-word and 25-word sentences)
- Use informal transitions sometimes ("honestly", "to be fair")
- Make it sound like a DRAFT, not a polished final version
- Keep the same meaning and academic level
- Do NOT use: "furthermore", "moreover", "in conclusion", "it is important"

Paragraph:
{paragraph}
"""
```

- [ ] **إضافة خيار في الواجهة:** "إعادة صياغة بالذكاء الاصطناعي (أقوى)"
- [ ] **دعم مفاتيح API متعددة:** OpenAI, Google Gemini, Anthropic
- [ ] **Rate limiting:** حد أقصى للطلبات لتجنب التكلفة العالية
- [ ] **Fallback:** إذا فشل الـ API، يرجع للمحرك المحلي

#### 4.2 نظام اختبار تلقائي (Auto-Testing Pipeline)

- [ ] **بناء test suite** يحتوي على 20+ نص اختبار (AI و بشري)
- [ ] **مقارنة تلقائية:** بعد كل تعديل على المحرك، يشغّل الاختبارات ويقارن النتائج
- [ ] **تكامل مع ZeroGPT API** (إن وُجد) للاختبار الآلي

#### 4.3 تحسين واجهة المستخدم

- [ ] **مؤشر "قبل/بعد" بصري** لكل فقرة مع تظليل التغييرات
- [ ] **خيار "مستوى الأنسنة":** خفيف (حفظ المعنى) → متوسط → عدواني
- [ ] **معاينة مباشرة** قبل التحميل
- [ ] **تاريخ التعديلات** — إمكانية التراجع
